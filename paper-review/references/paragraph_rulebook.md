# 段落级审阅规则库（通用底盘）

本文件为段落精审提供通用框架，适用于所有段落类型。专项规则（引言/综述/方法/分析/结论/事实）优先于通用规则，但通用维度可作为补充检查。

---

## 通用审阅原则

1. **先识别段落功能，再评价质量**
2. **把局部修改链接到全局论题**
3. **因段落类型而异的审查标准**
4. **先结构后表达**

---

## 段落类型识别系统

### 9 种段落类型

| 类型 | 编号 | 核心识别线索 | 路由规则 |
|---|---|---|---|
| 问题提出型 | P1 | "本文聚焦""试图回答""研究旨在" | → intro |
| 背景铺陈型 | P2 | "近年来""随着……发展" | → intro / fact |
| 概念界定/理论引入型 | P3 | "本文所指""在……理论中""框架" | → literature + 通用 |
| 事实性描述型 | P4 | "数据显示""截至""政策规定" | → fact |
| 综述性描述型 | P5 | "已有研究""学者认为""文献表明" | → literature |
| 方法说明型 | P6 | "本文采用""样本选取""数据来源" | → method |
| 分析论证型 | P7 | "这表明""其原因在于""进一步看" | → analysis |
| 结论/建议型 | P8 | "综上""本文认为""建议""局限" | → conclusion |
| 过渡/承接型 | P9 | "基于上述""在此基础上""下面将" | 通用规则 |

### 识别规则

1. 一个段落可同时包含两种功能 → 以主导功能为主类型
2. 过渡段不能是大段 → 单独占大篇幅的"过渡段"可能是伪过渡
3. 识别后才审查 → 先确定段落在做什么，再判断做得好不好

## 通用十维审查框架

以下 10 个维度适用于任何段落类型：

### D1：功能清晰度
本段在全文中承担什么功能？功能是否可识别？

### D2：中心明确度
本段的中心意思是否一句话可概括？

### D3：上下文匹配度
本段是否出现在合理位置？是否与前后文衔接？

### D4：信息质量
事实、数据、引用是否准确、可信、可追溯？

### D5：论证强度
（适用于分析段）主张是否有证据支撑？推理是否成立？

### D6：解释充分度
本段是否只停留在"是什么"？是否追问了"为什么"？

### D7：理论关联度
（适用于分析段）是否与全文理论框架/概念系统对应？

### D8：表达清晰度
句子是否通顺、无歧义、无冗余？

### D9：格式规范度
引用格式、图表标注、术语使用是否规范？

### D10：服务主线度
本段是否服务于论文核心问题？是否可被结论吸收？

## 段落审阅输出格式

对每个审阅的段落，输出结构如下：

```
### 段落位置
[章节 + 段序号]

### 段落类型
[从9种类型中识别]

### 当前功能
[一句话描述本段在做什么]

### 优点
[肯定已做到的部分]

### 问题
[按通用维度 + 专项维度检查，列出问题]

### 修改建议
[具体可执行的修改动作]

### 动作标签
[如：补中心句 / 压缩背景 / 补证据 / 补来源]

### 短批注
[一到两句建设性评语]
```
