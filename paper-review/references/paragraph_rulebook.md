# 段落级审阅规则库（通用底盘）

本文件为段落精审提供通用框架，适用于所有段落类型。专项规则（引言/综述/方法/分析/结论/事实）优先于通用规则，但通用维度可作为补充检查。

---

## 通用审阅原则

1. **先识别段落功能，再评价质量** —— 不知道一段在做什么，就无法判断做得好不好
2. **把局部修改链接到全局论题** —— 段落改得再好，如果不服务于主线，也是无效修改
3. **因段落类型而异的审查标准** —— 事实段要查来源，分析段要查证据，综述段要查组织，不能一刀切
4. **先结构后表达** —— 段落的功能和位置比措辞更重要

## 段落类型识别系统

### 9 种段落类型

| 类型 | 编号 | 核心识别线索 | 路由规则 |
|---|---|---|---|
| 问题提出型 | P1 | "本文聚焦""试图回答""研究旨在""拟探讨" | → intro |
| 背景铺陈型 | P2 | "近年来""随着……发展""在……背景下""当前……面临" | → intro / fact |
| 概念界定/理论引入型 | P3 | "本文所指""在……理论中""框架""界定为""概念源于" | → literature + 通用 |
| 事实性描述型 | P4 | "数据显示""截至""政策规定""根据……统计""表X显示" | → fact |
| 综述性描述型 | P5 | "已有研究""学者认为""文献表明""研究主要集中于""争议在于" | → literature |
| 方法说明型 | P6 | "本文采用""样本选取""数据来源""通过……方法""筛选标准" | → method |
| 分析论证型 | P7 | "这表明""其原因在于""进一步看""由此可见""从……维度看" | → analysis |
| 结论/建议型 | P8 | "综上""本文认为""建议""局限""未来可进一步" | → conclusion |
| 过渡/承接型 | P9 | "基于上述""在此基础上""下面将""接下来""为进一步" | 通用规则 |

### 识别规则

1. **一段可同时包含两种功能** → 以主导功能为主类型，次要功能触发叠加规则
2. **过渡段不能是大段** → 单独占大篇幅的"过渡段"可能是伪过渡，应重新识别其真实功能
3. **识别后才审查** → 先确定段落在做什么，再判断做得好不好
4. **注意"标签≠功能"** → 段落标题写着"分析"但实际在描述事实，应按事实段处理
5. **短段落合并判断** → 连续 2-3 个短段落可能共同构成一个功能单元，一起识别

## 通用十维审查框架

以下 10 个维度适用于任何段落类型。对不同类型段落，各维度的权重不同（标注了主要适用对象）。

### D1：功能清晰度
- **问题**：本段在全文中承担什么功能？功能是否可识别？
- **适用**：所有段落
- **检查**：能否用一句话说清"这一段是用来做什么的"
- **风险**：段落无明确功能 / 段落做的事和标题不一致 / 段落试图同时做太多事
- **动作**：明确段落功能 / 拆分多功能段落 / 删除无功能段落

### D2：中心明确度
- **问题**：本段的中心意思是否一句话可概括？
- **适用**：所有段落
- **检查**：是否有中心句 / 中心句是否位于段首或段末 / 其余句子是否围绕中心展开
- **风险**：段落冗长但无中心 / 中心句被淹没 / 一段话两个中心
- **动作**：补写中心句 / 将中心句前置 / 拆分双中心段落

### D3：上下文匹配度
- **问题**：本段是否出现在合理位置？是否与前后文衔接？
- **适用**：所有段落
- **检查**：是否承接前文 / 是否为后文做铺垫 / 前后段是否存在逻辑跳跃
- **风险**：段落孤立 / 与前后文主题突然切换 / 缺少过渡句
- **动作**：补过渡句 / 调整段落位置 / 补充桥梁句

### D4：信息质量
- **问题**：事实、数据、引用是否准确、可信、可追溯？
- **适用**：P2 背景、P4 事实、P5 综述（高权重）；其他类型（基本检查）
- **检查**：来源是否明确 / 数据是否最新 / 是否把推断写成事实 / 是否"据相关资料显示"
- **风险**：无来源事实 / 数据过旧 / 来源与内容不匹配
- **动作**：补来源 / 更新数据 / 降低判断强度

### D5：论证强度
- **问题**：主张和证据之间是否匹配？推理是否成立？
- **适用**：P7 分析（核心维度）；P8 结论（重要维度）
- **检查**：主张是否有证据 / 证据是否直接支撑主张 / 推理链是否完整
- **风险**：主张无证据 / 证据弱但结论强 / 推理跳跃
- **动作**：补证据 / 降低结论强度 / 补推理环节

### D6：解释充分度
- **问题**：本段是否只停留在"是什么"？是否追问了"为什么"？
- **适用**：P7 分析（核心维度）；P4 事实（需要时补充）
- **检查**：有现象但没有原因 / 有数据但没有解读 / "说明了""反映出"但没展开
- **风险**：描述冒充分析 / 机械复述图表 / 回避因果追问
- **动作**：增加原因分析 / 增加"为什么"解释 / 增加条件说明

### D7：理论关联度
- **问题**：是否与全文理论框架/概念系统对应？
- **适用**：P7 分析（核心维度）；P3 概念/理论（重要维度）
- **检查**：分析是否使用了前文理论维度 / 结论是否回扣理论框架
- **风险**：理论只在综述出现，分析完全脱节 / 理论贴标签不真正使用
- **动作**：补理论连接句 / 用理论维度重组分析

### D8：表达清晰度
- **问题**：句子是否通顺、无歧义、无冗余？
- **适用**：所有段落
- **检查**：是否有病句 / 是否有歧义 / 术语是否统一 / 指代是否明确
- **风险**：长句嵌套 / 主语缺失 / 术语混用 / "这""其"指代不清
- **动作**：拆分长句 / 补主语 / 统一术语 / 明确指代

### D9：格式规范度
- **问题**：引用格式、图表标注、术语使用是否规范？
- **适用**：所有段落
- **检查**：引用格式 / 图表编号与来源 / 数字单位 / 脚注/尾注
- **风险**：格式不统一 / 图表无来源 / 正文与图表数据不一致
- **动作**：统一格式 / 补来源 / 核对数据一致性

### D10：服务主线度
- **问题**：本段是否服务于论文核心问题？是否可被结论吸收？
- **适用**：所有段落
- **检查**：段落内容是否与研究问题有关 / 删去此段后论证链是否断裂
- **风险**：段落内容正确但与主线无关 / 大量篇幅不服务于核心问题
- **动作**：删除偏题内容 / 增加回扣主线的句子 / 缩减并整合

## 各类型段落审查重点速查

| 段落类型 | 高权重维度 | 关键审查点 |
|---|---|---|
| P1 问题提出 | D1, D2, D10 | 问题是否明确、是否可研究、边界是否清楚 |
| P2 背景铺陈 | D3, D4, D10 | 背景是否服务于问题、是否过长、是否有桥梁句 |
| P3 概念/理论 | D4, D7, D2 | 概念是否准确、理论是否与本文匹配、是否形成分析维度 |
| P4 事实描述 | D4, D6, D9 | 事实是否准确、来源是否明确、是否有解释 |
| P5 综述描述 | D4, D6, D10 | 是否按主题组织、是否有评述、是否引出研究位置 |
| P6 方法说明 | D1, D4, D6 | 方法是否匹配问题、流程是否透明、对象是否明确 |
| P7 分析论证 | D5, D6, D7 | 主张是否有证据、是否有解释、是否与理论关联 |
| P8 结论/建议 | D5, D10, D2 | 是否回应问题、边界是否合理、建议是否可操作 |
| P9 过渡/承接 | D3, D1 | 是否有效连接前后文、是否过长 |

## 段落审阅输出格式

对每个审阅的段落，输出结构如下：

```
### 段落位置
[章节 + 段序号]

### 段落类型
[从9种类型中识别]

### 当前功能
[一句话描述本段在做什么]

### 优点
[肯定已做到的部分]

### 问题
[按通用维度 + 专项维度检查，列出问题]

### 修改建议
[具体可执行的修改动作]

### 动作标签
[如：补中心句 / 压缩背景 / 补证据 / 补来源 / 重写 / 拆分 / 合并 / 前移 / 后移 / 删除]

### 短批注
[一到两句建设性评语]
```
