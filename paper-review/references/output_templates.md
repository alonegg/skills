# 输出标准模板库

本文件为论文审阅的各级输出提供标准模板。所有审阅报告应参照此处模板输出，确保格式一致、信息完整、层次清晰。

---

## 原则

1. **分层输出**：全文→章节→段落，逐级下沉
2. **问题—影响—动作链**：每个重要问题都包含"是什么""有什么影响""怎么改""改到什么程度"
3. **粒度控制**：全文报告控制全局，段落报告控制细节，不交叉
4. **输出即输入**：本轮输出是下一轮复审的输入

---

## Template 1：全文审阅报告

```markdown
# 全文审阅报告

## 论文画像
- 题目：
- 论文类型：[理论分析/实证研究/案例研究/政策分析/应用设计]
- 研究对象：
- 核心研究问题：
- 主要方法：
- 核心观点/发现：
- 结构概览：[各章标题与功能]

## 总体评价
[2-3句概括论文当前状态、主要优点、最关键问题]

## 全局关键问题（3-5个）

### 问题1：[标题]
- 级别：S/A/B/C
- 位置：[涉及章节]
- 描述：
- 影响：
- 修改动作：
- 修改目标：

### 问题2-5：[同上结构]

## 结构与章节诊断
[对各章功能的逐一判断]

## 修订优先级
1. [第一优先修改项]
2. [第二优先修改项]
3. ...

## 下一轮修改重点
[建议学生先解决什么，才能有效推进后续修改]
```

## Template 4：修订任务单

```markdown
# 修订任务单

## 概要
- 审阅轮次：第N轮
- 发现问题总数：
- S级：N个 / A级：N个 / B级：N个 / C级：N个
- 建议本轮优先处理：S级和A级

## 任务列表

| 编号 | 级别 | 位置 | 类型 | 描述 | 修改动作 | 修改目标 | 前置依赖 | 状态 |
|---|---|---|---|---|---|---|---|---|
| T1 | S | 引言 | 问题不清 | 核心研究问题未明确 | 在引言末明确问题句 | 读者可一句话知道研究什么 | 无 | PENDING |
| T2 | A | 综述 | 位置缺失 | 综述未引出研究位置 | 段末补研究空白和切口 | 综述与本文形成逻辑连接 | T1 | PENDING |
| ... |

## 修改顺序建议
1. 先处理 T1（S级，无依赖）
2. 再处理 T2（A级，依赖T1）
3. ...
```

```markdown
## 新发现问题

## 本轮总结
- 已解决：N个
- 部分解决：N个
- 未解决：N个
- 新问题：N个

## 下一步建议
[建议下一轮重点]
```

## Template 6：短批注列表

用于快速审阅或针对段落级的简短反馈：

```markdown
| 位置 | 类型 | 批注 |
|---|---|---|
| 引言§2 | 背景控制 | "背景铺陈较充分，但与核心问题的连接仍可更紧。" |
| 综述§4 | 评述能力 | "目前仍偏作者罗列，建议改按主题组织。" |
| 分析§2 | 主张清晰度 | "本段材料较充分，但中心判断仍可再明确。" |
```

## Template 7：审核意见文档（Word 格式）

用于生成独立的审核意见报告 Word 文档（配合 Word 文档审阅工作流使用）。
使用 docx-js 生成，结构如下：

```
标题：《论文审核意见》
副标题：论文题目 + 审阅日期 + 审阅轮次

一、论文画像
  - 题目 / 类型 / 研究对象 / 核心问题 / 主要方法 / 结构概览

二、总体评价
  - 2-3 句概括（优点 + 最关键问题）

三、全局关键问题
  - 表格：编号 | 级别 | 位置 | 问题描述 | 影响 | 修改建议 | 修改目标

四、修订任务单
  - 表格：编号 | 级别 | 位置 | 类型 | 描述 | 动作 | 目标 | 依赖 | 状态

五、章节诊断
  - 每章一段：功能判断 + 主要问题 + 修改方向

六、修改优先级与建议
  - 按优先级排列的修改路线图

七、下一轮审阅重点
  - 建议学生先解决什么问题
```

格式要求：
- 标题使用 Heading 1，各部分使用 Heading 2
- 问题表格使用带边框的表格，S 级行加红色底色，A 级行加橙色底色
- 页眉含论文题目，页脚含页码
