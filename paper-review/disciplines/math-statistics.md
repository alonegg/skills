# 数理统计类 学科配置

> 本文件是 paper-review 技能的学科叠加层。它不替换通用规则，而是在通用规则基础上增加数理统计类学科特有的审查维度、调整权重、补充常见问题模式。

## 适用范围

- **覆盖学科**：数学与应用数学、信息与计算科学、统计学、应用统计学、数据科学与大数据技术、精算学
- **典型论文类型**：
  - **理论证明型**：提出或推广数学定理，给出完整证明（如推广某不等式到更一般的空间、证明某类方程解的存在唯一性）
  - **应用建模型**：针对实际问题建立数学模型并求解（如传染病传播模型、金融衍生品定价模型、排队论模型）
  - **统计分析型**：使用统计方法对实际数据进行假设检验、回归分析、因子分析等，得出统计推断结论
  - **数值计算型**：设计或改进数值算法求解数学问题（如偏微分方程数值解、大规模线性方程组迭代求解、优化算法设计）
  - **数据分析型**：基于大规模数据集进行探索性分析、预测建模、聚类分析等，侧重数据驱动的方法
- **识别线索**：论文中出现定义-定理-证明链、数学公式推导占大量篇幅、统计检验表格（p值、置信区间）、收敛性分析、误差估计、算法伪代码配合数值实验结果等

## 结构期望

### 标准章节结构

本学科论文结构因论文类型而异，审查时应先判断论文属于哪种类型，再对照对应结构：

**理论证明型**：
```
第1章 绪论（研究背景、问题来源、国内外研究现状、本文贡献）
第2章 预备知识（基本定义、已知引理/定理、符号约定）
第3章 主要结果（定理陈述与完整证明）
第4章 应用与实例（主要结果的应用或特殊情形）
第5章 总结与展望
```

**应用建模型**：
```
第1章 绪论（问题背景、建模动机、研究现状）
第2章 模型建立（假设条件、变量定义、模型推导）
第3章 模型求解与分析（解析解/数值解、稳定性分析、灵敏度分析）
第4章 实例验证或数值模拟（模型与实际数据的拟合验证）
第5章 总结与展望
```

**统计分析型**：
```
第1章 绪论（研究问题、研究意义、研究现状）
第2章 理论基础与方法介绍（统计方法原理、适用条件）
第3章 数据来源与预处理（数据采集、清洗、描述性统计）
第4章 实证分析（假设检验、回归分析、模型诊断）
第5章 结果讨论（统计结论的实际含义、与已有研究的比较）
第6章 总结与展望
```

**数值计算型**：
```
第1章 绪论（问题来源、计算困难、已有算法综述）
第2章 预备知识（数学模型、已有算法回顾）
第3章 算法设计（算法描述、收敛性分析、复杂度分析）
第4章 数值实验（测试问题选取、精度对比、效率对比、参数影响）
第5章 总结与展望
```

**数据分析型**：
```
第1章 绪论（研究背景、研究问题）
第2章 文献综述与方法介绍
第3章 数据采集与预处理（数据来源、缺失值处理、特征工程）
第4章 模型建立与分析（方法选择、模型训练、交叉验证）
第5章 结果与讨论
第6章 总结与展望
```

### 必须有的章节

- **绪论**：必须包含明确的数学问题陈述（本文要解决什么数学/统计问题），不能只有空泛的"数学在生活中很重要"
- **预备知识/理论基础**：理论证明型和数值计算型论文必须有此章节，包含所有后续推导所需的定义、引理及其出处
- **核心工作章节**：理论型必须有完整的定理证明；建模型必须有模型建立与求解；统计型必须有实证分析；数值型必须有算法设计与数值实验
- **验证/实验章节**：建模型需有模型验证（与实际数据对比或与已知结果对比）；数值型需有数值实验；统计型需有模型诊断
- **总结与展望**：必须基于本文实际工作进行数学层面的总结，而非空泛评价

### 可选/特殊章节

- **符号表/记号约定**：当论文符号系统复杂时，建议设独立的符号表（可作为附录或预备知识章节一部分）
- **附录（详细证明）**：主要定理的技术性引理证明、长计算推导可放入附录，但主定理的证明必须在正文
- **程序代码附录**：数值计算型论文的核心算法代码可作为附录提供

### 结构红线

- **理论型论文无预备知识章节**：主要结果依赖的定义和引理散落在证明中临时引入，读者无法理顺逻辑前提
- **证明缺失或不完整**：声称证明了定理但只给出证明思路或"证明略"，对于本科论文的主要结果不可接受
- **统计分析型无模型诊断**：建立了回归模型但未进行残差分析、多重共线性检验、异方差检验等诊断
- **数值实验无对比基准**：只展示了本文算法的结果，没有与已有算法或解析解进行对比

## 专项规则权重调整

基于通用6大专项规则，调整本学科的审查权重：

| 专项规则 | 权重调整 | 理由 |
|---|---|---|
| intro | 0 | 数学类论文绪论通常较简洁，重点在问题陈述的精确性而非篇幅 |
| literature | +1 | 需要准确梳理问题的研究历史，区分已解决和未解决的部分 |
| method | +2 | 数学方法（证明/建模/算法设计/统计方法选择）是论文核心，需最严格审查 |
| analysis | +2 | 证明的严谨性、数值结果的可靠性、统计推断的正确性是论文质量的决定性因素 |
| conclusion | 0 | 无特殊调整 |
| fact | +1 | 数学论文中的事实性陈述（已知定理、公式引用）必须准确无误，错误引用直接影响后续推导 |

## 方法审查补充维度

通用 method 规则之外，本学科需额外检查：

### 数学严谨性审查

1. **定义-定理-证明链完整性**：每个定理是否有完整证明？证明中引用的每个引理/定义是否已在预备知识中给出或标注了出处？是否存在"显然"、"容易看出"、"易证"等跳步用语掩盖非平凡推导？
   - 检查要点：证明中的每一步是否有逻辑依据（前文定理/引理编号引用、已知不等式、定义展开等）
   - 特别关注：反证法是否得到了矛盾、数学归纳法是否验证了基础步和归纳步、构造性证明是否验证了所构造对象满足所有条件

2. **条件与假设显式陈述**：定理的条件是否完整列出？是否存在隐含假设未声明的情况？条件是否为充分条件、必要条件还是充要条件——是否标注清楚？
   - 检查要点：函数的可微性/连续性/可积性假设、空间的完备性/紧致性假设、矩阵的正定性/可逆性假设、随机变量的独立性/分布假设

3. **逻辑间隙检测**：证明中是否存在逻辑跳跃？从A推出B的论证是否充分？是否混淆了充分条件和必要条件？
   - 检查要点：量词使用（"存在"与"任意"不可混淆）、否命题与逆命题的区别、反例对条件必要性的说明

### 模型建立规范

4. **模型假设合理性**：模型的每个假设是否明确陈述？假设是否有现实依据或数学必要性的论证？假设的合理性是否进行了讨论？
   - 检查要点：假设是否过强导致模型缺乏实用性？假设是否过弱导致问题不可解？是否与参考文献中类似模型的假设进行了对比？

5. **变量与参数定义**：所有变量是否在首次出现时给出明确定义？参数的取值范围和物理/统计含义是否说明？模型的因变量和自变量是否清晰区分？
   - 检查要点：同一字母是否在不同上下文中表示不同含义、参数单位是否匹配、维度分析是否一致

6. **模型局限性讨论**：模型在什么条件下有效？模型简化了哪些实际因素？这些简化对结论有何影响？参数变化对结论的敏感度如何？
   - 检查要点：是否进行了灵敏度分析/参数敏感性分析、是否讨论了模型的适用边界

### 统计方法选择审查

7. **分布假设检验**：使用参数方法前是否验证了分布假设？正态性检验（Shapiro-Wilk、K-S检验）是否执行？方差齐性检验（Levene检验、Bartlett检验）是否在方差分析前完成？
   - 检查要点：样本量是否满足方法要求、是否在假设不满足时采用了替代方法（如非参数检验）

8. **样本量论证**：样本量是否足以支撑所用统计方法？是否进行了功效分析（power analysis）或给出了样本量选择的理由？
   - 检查要点：回归分析中样本量与自变量个数的比例、小样本情况下是否使用了适当的修正方法

9. **多重比较校正**：同时进行多个假设检验时是否进行了多重比较校正（Bonferroni校正、FDR控制等）？
   - 检查要点：多重检验未校正会导致整体I类错误率膨胀，p < 0.05的结论可能不可靠

10. **方法假设核实**：线性回归是否检验了线性性、独立性、正态性、等方差性？logistic回归是否检验了样本量充足性和多重共线性？时间序列是否检验了平稳性？
    - 检查要点：每种统计方法都有适用前提，使用前必须逐一验证

### 数值实验设计审查

11. **算法收敛性分析**：迭代算法是否给出了收敛性的理论证明或数值验证？收敛速度是否与理论预期一致？是否给出了收敛阶数的数值估计？
    - 检查要点：迭代次数-误差关系图、Richardson外推法验证收敛阶

12. **误差估计与界**：数值方法的截断误差和舍入误差是否分析？误差界是否给出？数值解与精确解（如已知）的误差是否报告？
    - 检查要点：网格细化时误差的下降趋势是否符合理论阶数

13. **测试问题选取**：数值实验选取的测试问题是否具有代表性？是否包含了方法适用和不适用的情形？是否包含了已知精确解的检验问题？
    - 检查要点：测试问题不能只选对本文算法有利的简单情形

14. **计算复杂度验证**：理论复杂度分析是否与实际运行时间一致？不同问题规模下的运行时间变化趋势是否符合渐进复杂度？

### 数据处理规范审查

15. **数据清洗过程**：数据清洗的具体步骤是否详细记录？删除或修改了多少数据？清洗标准是什么？
    - 检查要点：数据清洗不能是黑箱操作，必须可追溯可复现

16. **缺失值处理**：缺失值的比例是多少？采用了何种填补方法（删除/均值填补/多重填补/EM算法）？为什么选择该方法？缺失机制（MCAR/MAR/MNAR）是否讨论？
    - 检查要点：大量删除缺失数据可能引入选择偏倚

17. **异常值处理**：异常值的判定标准是什么（IQR规则、3sigma规则、Cook距离）？处理方式（删除/缩尾/Winsorize）是否合理？是否分析了异常值对结果的影响？

18. **证明技巧专项审查**：数学归纳法是否完整验证了基础步和归纳步？epsilon-delta论证是否严格？存在性和唯一性是否分别证明？构造性证明是否验证了所构造对象满足所有所需性质？

## 证据与论证标准补充

通用 analysis/fact 规则之外，本学科的证据标准：

### 可接受的证据类型

- **完整数学证明**：对于理论性结论，完整的逻辑推导是最高级别的证据。证明必须无逻辑缺口，每一步有明确依据
- **数值实验结果**：与精确解的误差对比、不同网格/步长下的收敛性实验、与已有算法的效率对比。需列出实验参数和计算环境
- **统计检验结果**：假设检验的检验统计量、p值、置信区间、效应量。必须同时报告统计显著性和实际显著性
- **模型验证数据**：模型预测值与实际观测值的对比、拟合优度指标（R-squared、AIC、BIC）、残差分析图
- **反例**：用于证明条件必要性或否定某个猜想，一个严格的反例即为充分证据
- **仿真实验**：Monte Carlo模拟验证理论概率结果、Bootstrap抽样验证估计量的分布性质，需说明模拟次数和随机种子

### 证据红线

- **"显然"掩盖非平凡推导**：用"显然"、"易证"、"不难看出"跳过关键推导步骤——如果这一步真的显然，那就用一两句话写出来；如果不显然，必须完整证明
- **证明举例法**：用若干具体数值验证来"证明"一般性命题——具体例子只能验证不能证明，这是数学论文的根本错误
- **统计结论无前提验证**：直接报告t检验p值但未验证正态性假设、直接使用Pearson相关系数但未检验线性关系——方法前提不满足时结论无效
- **p值万能论**：仅凭p < 0.05就声称"显著"而不报告效应量和置信区间——统计显著性不等于实际显著性
- **过拟合的模型声称**：训练集R-squared极高但未报告测试集表现，或未进行交叉验证——模型预测能力不明
- **数值精度无参照**：称"算法精度达到10^-6"但不说明参照的精确解是什么、精度如何计算——绝对数字无意义
- **选择性报告**：只报告模型表现好的指标或数据子集，回避模型失败的情形

### 论证模式期望

- **理论证明型**：定义→引理→定理→证明→推论/应用的严格演绎链。每个步骤必须有逻辑依据，证明结束必须有QED标记或证明环境标识
- **应用建模型**：问题抽象→假设建立→模型推导→模型求解→模型验证→灵敏度分析的完整链路。必须有假设到结论的因果逻辑
- **统计分析型**：研究假设→方法选择（含前提验证）→统计推断→结果解释→局限性讨论的实证研究范式。结论不能超出数据和方法所支撑的范围
- **数值计算型**：问题描述→算法设计→理论分析（收敛性/稳定性/复杂度）→数值验证→与已有方法对比的理论-实验互证范式
- **数据分析型**：数据描述→探索性分析→方法选择→模型建立→结果验证→结论解释的数据驱动范式。探索性分析的发现应指导后续建模方向

## 常见问题模式

### S级（论文是否成立）

1. **定理陈述无证明或证明存在关键逻辑缺口**：
   - 表现：论文的主要定理仅给出陈述但标注"证明略"或"证明思路参见文献[X]"；或证明中存在关键步骤无法衔接，中间跳跃了一个非平凡的推导
   - 影响：论文的核心学术贡献不成立，主要结果的正确性无法确认
   - 修改方向：补全所有主要定理的完整证明；对于确实可省略的辅助引理证明，可移至附录但必须完整给出；关键跳步处需逐步展开推导

2. **统计结论建立在未验证的方法假设之上**：
   - 表现：对明显非正态的数据直接使用t检验或方差分析并得出显著性结论；对存在严重异方差的数据直接使用最小二乘回归；对非平稳时间序列直接建立ARMA模型
   - 影响：所有统计推断可能无效，论文的核心数据分析结论不可信
   - 修改方向：在使用每种统计方法之前补充前提假设的检验步骤；假设不满足时改用替代方法（如非参数检验、加权最小二乘、ARIMA模型）

3. **模型建立在未声明或不合理的假设之上**：
   - 表现：模型推导过程中隐含使用了某些假设（如独立性、线性性、连续可微性）但未明确说明；或假设明显与实际问题矛盾（如对离散数据假设连续分布、对相关数据假设独立性）
   - 影响：模型的有效性缺乏理论基础，所有基于模型的结论可能错误
   - 修改方向：逐一列出所有假设条件并论证其合理性；对于简化假设，必须讨论其对结论的影响程度；如假设不合理，需修改模型

4. **证明举例法——以特例代替证明**：
   - 表现：对一般性命题仅用若干具体数值代入验证（如"当n=1,2,3时命题成立，因此命题对一切n成立"），没有给出严格的一般性证明
   - 影响：论文声称证明的结论实际上未被证明，这是数学论文中最根本的逻辑错误
   - 修改方向：必须给出严格的一般性证明（如数学归纳法、反证法等）；具体例子只能作为辅助说明，不能替代证明

### A级（严重影响质量）

1. **定理条件陈述不完整**：
   - 表现：定理中缺少关键前提条件（如未说明函数的可微性阶数、未限定参数取值范围、未要求矩阵的正定性），导致定理严格来说是错误的
   - 影响：定理的适用范围不明确，读者可能在不满足条件的情形下误用结论
   - 修改方向：对照参考文献中类似定理的条件陈述，逐一补全所有必要条件；用反例说明某些条件的必要性

2. **缺少灵敏度分析/参数敏感性分析**：
   - 表现：建模型论文确定了参数值后直接给出结论，未分析参数变化对结论的影响；或统计模型中自变量的微小变化导致结论剧烈变化但未讨论
   - 影响：读者无法评估结论的稳健性，模型可能在参数轻微变化时失效
   - 修改方向：对关键参数进行灵敏度分析，给出参数变化范围与结论变化的定量关系；讨论哪些参数对结果影响最大

3. **统计模型缺少拟合优度评估**：
   - 表现：建立了回归模型或其他统计模型但未报告拟合优度指标（R-squared、调整R-squared、AIC/BIC）；或未进行残差分析以检验模型适当性
   - 影响：模型的解释力和预测力不明，可能存在严重的模型误设
   - 修改方向：补充拟合优度指标；进行残差图分析（残差vs拟合值、Q-Q图、残差自相关图）；如发现模型不适当，需调整模型形式

4. **数值实验缺少收敛性验证**：
   - 表现：数值算法只在一组参数下展示了计算结果，未通过网格细化/步长缩小来验证数值解的收敛性和精度阶
   - 影响：无法确认数值结果是否可靠，算法的理论收敛性未得到实验验证
   - 修改方向：设计网格细化实验，给出误差随步长变化的对数图，验证收敛阶数是否与理论分析一致

5. **充分条件与必要条件混淆**：
   - 表现：证明了A→B后声称B→A也成立但未证明；或将充分条件当作充要条件使用；在应用中将必要条件当作充分条件进行判定
   - 影响：逻辑推导错误，结论的范围被不当扩大或缩小
   - 修改方向：严格区分充分条件、必要条件和充要条件；如需充要条件，必须分别证明两个方向；修正论文中的逻辑表述

6. **统计显著性替代实际显著性**：
   - 表现：仅报告p < 0.05即声称变量"有显著影响"，不报告效应量（Cohen's d、eta-squared等）和置信区间；或样本量极大时将微小差异解读为"显著"
   - 影响：读者无法判断统计差异是否具有实际意义，容易得出误导性结论
   - 修改方向：统计检验结果必须同时报告效应量和置信区间；对效应量的实际意义进行讨论；大样本时特别注意区分统计显著与实际显著

### B级（影响说服力）

1. **符号使用不一致**：
   - 表现：同一符号在不同章节表示不同含义（如x在第2章表示自变量，在第4章表示样本值）；或同一概念使用不同符号（如方差既用sigma^2又用Var(X)且未说明一致性）
   - 影响：增加读者理解负担，可能导致理解歧义，暴露写作不严谨
   - 修改方向：统一全文符号系统，建立符号表；同一概念全文使用同一符号；不同概念避免使用相同字母

2. **应用问题中缺少单位标注**：
   - 表现：模型中的变量有物理/经济含义但未标注单位（如"设某商品价格为p"但未说明单位是元还是万元）；公式两侧量纲不匹配
   - 影响：结论的实际含义不明确，量纲不一致可能隐藏计算错误
   - 修改方向：所有具有实际含义的变量标注单位；检查公式两侧量纲一致性；结论中的数值配合单位呈现

3. **图表缺少必要标注**：
   - 表现：数值实验图缺少坐标轴标签或单位、散点图缺少图例说明、统计图缺少误差棒/置信区间标注、收敛性图的坐标轴未使用对数刻度导致趋势不明显
   - 影响：读者无法准确读取图表信息，降低论文的专业性
   - 修改方向：所有图表必须包含标题、坐标轴标签（含单位）、图例；数值实验的误差收敛图建议使用对数坐标；统计图附上误差信息

4. **引用"众所周知的结果"但未给出出处**：
   - 表现："由泛函分析中的经典结论可知……"、"根据概率论的基本定理……"但未标注具体是哪个定理、出自哪本教材或文献
   - 影响：读者无法验证引用的正确性，也无法查阅原始来源进一步了解
   - 修改方向：所有引用的定理/引理/公式给出明确出处（文献编号+定理编号或页码）；即使是教材中的结论也需标注具体教材和版本

5. **数值计算代码不可复现**：
   - 表现：数值实验结果无法复现，因为论文未提供核心算法代码、未说明编程语言和软件版本、未给出随机种子或计算参数的完整设置
   - 影响：数值结果的可信度降低，其他研究者无法验证或扩展
   - 修改方向：在附录中提供核心算法的代码实现；正文中列出完整的计算参数表（步长、迭代次数、容许误差、随机种子等）；注明计算环境（软件版本、硬件配置）

6. **公式编号与引用不规范**：
   - 表现：论文中部分公式有编号部分没有，正文中引用公式时使用"上式"、"前面的公式"而非编号引用，导致跨页阅读时无法定位
   - 影响：降低论文的可读性和学术规范性
   - 修改方向：所有被正文引用的公式必须编号；引用公式时使用"由式(2.3)可得"而非"由上式可得"；编号采用"章节号.公式序号"格式

## 文献综述特殊要求

### 文献来源优先级

1. **国际顶级期刊**：Annals of Mathematics、Inventiones Mathematicae、Acta Mathematica、Journal of the American Mathematical Society (JAMS)、Annals of Statistics、Journal of the American Statistical Association (JASA)、Biometrika、Journal of the Royal Statistical Society (Series B)
2. **国际高水平期刊**：SIAM系列期刊（SIAM Review、SIAM Journal on Numerical Analysis、SIAM Journal on Optimization等）、Mathematics of Computation、Numerische Mathematik、Journal of Computational Physics、Statistical Science
3. **国内权威期刊**：数学学报、中国科学:数学、应用数学学报、数学年刊、统计研究、数理统计与管理、应用概率统计
4. **经典教材与专著**：Springer GTM/UTM系列、科学出版社研究生教材系列、人民教育出版社经典教材等，作为基础理论的引用来源
5. **预印本**：arXiv预印本可用于引用最新进展，但需注明为预印本，不能作为唯一文献来源

### 文献数量底线

- 理论证明型：至少15篇学术文献（期刊论文+专著），其中与本文主题直接相关的核心文献不少于5篇
- 应用建模型：至少15篇，包含应用领域文献和数学方法文献两部分
- 统计分析型：至少12篇，统计方法文献和应用领域文献均需涵盖
- 数值计算型：至少15篇，需涵盖同类算法的主要文献
- 数据分析型：至少12篇，方法论文献和领域文献均需涵盖

### 综述组织方式

- **禁止**：逐篇罗列式综述（"文献[1]研究了……，文献[2]研究了……"），这不是综述而是文献列表
- **要求**：按研究问题的发展脉络或方法分类组织，形成"问题起源→关键进展→当前前沿→尚存不足"的叙事线索
- **必须有**：综述末尾的"本文工作定位"段落，明确指出已有工作的不足或待解决的问题，以及本文的切入点和贡献
- **特别注意**：数学类论文的文献综述要重视问题的历史渊源，许多问题有数十年甚至上百年的研究历史，应追溯问题的提出者和关键推进者

## 术语与规范

### 公式排版规范

- **LaTeX质量排版**：所有公式必须使用LaTeX或同等质量的公式编辑器排版；手写公式拍照插入不可接受
- **标准数学符号**：使用标准数学符号（对应、存在量词、属于、子集、并集、交集、求和、积分等标准符号）
- **公式编号**：所有被引用的公式必须编号，采用"(章.序号)"格式，如(2.3)
- **变量定义**：公式中的每个新变量在首次出现时必须给出定义和含义说明
- **行间公式与行内公式**：重要公式（定理、核心推导步骤）使用行间公式单独成行；简单变量引用可使用行内公式
- **多行推导**：长推导使用对齐环境（如align），等号对齐，每行末尾标注推导依据

### 术语区分要求

- **充分条件/必要条件/充要条件**：严格区分，不可混用。陈述定理时必须明确标注条件类型
- **总体与样本**：区分总体参数（如总体均值mu）和样本统计量（如样本均值x-bar），不可混用符号
- **参数与统计量**：参数是总体的固定常数（如sigma），统计量是样本的函数（如S）；贝叶斯框架下参数可为随机变量但需明确说明
- **估计量与估计值**：估计量是随机变量（如样本均值X-bar），估计值是观测后的具体数值（如x-bar = 3.5）；论文中需根据上下文正确使用
- **收敛类型**：区分点态收敛/一致收敛、依概率收敛/几乎必然收敛/依分布收敛/L^p收敛，不可模糊处理
- **相关与因果**：统计分析中严格区分相关性（correlation）和因果性（causation），回归分析结论不可直接解读为因果关系

### 符号规范

- 所有非标准记号首次出现时必须定义，建议论文开头或预备知识章节设符号表
- 向量/矩阵使用粗体，标量使用斜体，集合使用花体或空心字母，遵循数学排版惯例
- 同一论文中保持符号体系一致：选定了某种记号风格（如概率用P还是Pr，期望用E还是E[·]）后全文统一

### 引用格式

- 遵循GB/T 7714-2015标准（中文学位论文通用要求）
- 期刊论文引用需包含：作者、论文标题、期刊名、年份、卷号、期号、页码
- 教材引用需包含：作者、书名、版次、出版社、出版年份
- 定理/引理引用格式：在正文中标注"由文献[X]中定理Y.Z可知"，具体到定理编号

## 结论与建议特殊要求

### 结论章节要求

- **成果精确表述**：理论型论文总结证明了哪些定理、推广了哪些已知结果、在什么意义上改进了前人工作；建模型论文总结模型的主要结论和适用条件；统计型论文总结主要统计发现及其实际含义；数值型论文总结算法的性能指标（精度阶、计算效率）与已有方法的对比结论
- **禁止空泛总结**："本文运用数学方法研究了XX问题，得到了一些有意义的结论"——这种结论完全没有信息量，必须具体说明是什么结论、有什么意义
- **条件与局限性重申**：结论中应重申主要结果的成立条件和适用范围，提醒读者不要超范围使用结论
- **开放问题指出**：对于理论型论文，如果主要结果的条件可以放宽但本文未完成，应作为开放问题明确指出

### 展望部分的边界

- **合理展望**：基于当前工作的已知局限提出改进方向，如"将定理X的条件从Hilbert空间推广到Banach空间"、"增加样本量以提高统计功效"、"改进算法以降低计算复杂度"
- **过度展望红线**：展望内容与本文工作无直接数学联系（如论文做的是微分方程数值解，展望中写"未来将研究量子计算在该领域的应用"）；展望过于宏观空泛（"本文结论对数学科学的发展具有重要意义"）
- **合理粒度**：展望应具体到可操作的数学问题或技术路线，如具体指出哪个条件可能可以放宽、哪个步骤可能可以优化、哪类数据可以进一步验证

## 学科专属审查维度

### 证明完整性专项检查

| 证明技巧 | 必须检查的要素 | 常见缺陷 |
|---|---|---|
| 数学归纳法 | 基础步验证、归纳假设陈述、归纳步推导 | 遗漏基础步或基础步验证不严格 |
| 反证法 | 假设的否命题、推导出矛盾、矛盾的明确指出 | 推导中引入了与否命题无关的额外假设 |
| 构造法 | 对象的显式构造、验证构造对象满足所有条件 | 构造了对象但未验证满足某些条件 |
| epsilon-delta论证 | delta的选取依赖于epsilon、不等式链完整 | delta的选取过程不清楚或依赖关系错误 |
| 压缩映射/不动点 | 映射的良定义性、压缩条件验证、空间完备性 | 未验证空间完备性或映射的自映射性质 |
| 极大极小方法 | 变分框架建立、紧性条件、PS条件验证 | 未验证Palais-Smale条件 |

### 统计报告规范检查

统计结果报告应包含以下要素（缺一项扣分）：

| 统计方法 | 必须报告的内容 | 常见缺漏 |
|---|---|---|
| 假设检验 | 原假设/备择假设、检验统计量值、p值、结论 | 只报告p值不报告检验统计量 |
| 置信区间 | 点估计、置信水平、区间上下界 | 只报告点估计不报告区间 |
| 回归分析 | 回归方程、各系数及其标准误、t值、p值、R-squared、F检验 | 只给出回归方程不报告显著性检验 |
| 方差分析 | ANOVA表（SS/df/MS/F/p）、事后多重比较结果 | 缺少事后比较或多重比较未校正 |
| 非参数检验 | 检验名称、检验统计量、p值、效应量 | 未说明选择非参数检验的理由 |

### 数值实验质量评估框架

**质量不足的信号**（多项命中则实验设计存疑）：
- 只在一组参数下展示结果，无参数敏感性分析
- 无已知精确解的检验问题验证
- 未报告收敛阶数或收敛阶数与理论不符但未讨论
- 仅与一种已有方法对比，且该方法明显过时
- 计算时间/复杂度完全未提及
- 测试问题全部为一维或低维简单问题

**质量充足的信号**（至少命中3-4项）：
- 包含已知精确解的检验问题并报告了精度
- 收敛阶数的数值验证与理论分析一致
- 与多种已有方法在相同问题上公平对比
- 包含不同维度/不同规模的测试问题
- 参数敏感性分析或鲁棒性测试完整
- 报告了算法的计算时间/内存占用并与问题规模的关系一致
- 讨论了算法失败或表现不佳的情形及原因
