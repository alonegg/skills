# 电子通信类 专项审阅配置

> 本文件为本科毕业论文审阅系统的学科叠加配置，在通用审阅规则之上增加电子通信类论文的专项审查维度。

---

## 一、适用范围

### 1.1 覆盖学科

- 电子信息工程
- 通信工程
- 信号与信息处理
- 电磁场与微波技术
- 集成电路设计
- 嵌入式系统
- 物联网工程

### 1.2 典型论文类型

| 类型 | 说明 | 识别线索 |
|------|------|----------|
| 系统设计型 | 完成一个电子/通信系统的设计与实现 | 含系统框图、模块划分、接口定义、整机联调 |
| 算法研究型 | 提出或改进信号处理/通信/图像算法 | 含算法流程图、复杂度分析、仿真对比曲线 |
| 仿真分析型 | 基于仿真工具对系统/电路/信道进行建模分析 | 大量仿真截图、参数扫描、蒙特卡洛分析 |
| 硬件实现型 | 完成电路板/FPGA/芯片的设计-制作-测试全流程 | 含原理图、PCB版图、实物照片、测试数据 |

### 1.3 识别线索

当论文中出现以下关键词集群时激活本配置：

- 硬件类：PCB、原理图、焊接、元器件、BOM、示波器、频谱仪、矢量网络分析仪
- 仿真类：MATLAB/Simulink、Multisim、HFSS、ADS、Cadence、Vivado、ModelSim、CST
- 信号类：采样率、频谱、滤波器、调制解调、信噪比、误码率、星座图
- 通信类：信道模型、OFDM、MIMO、编解码、均衡、同步、载波频偏
- 集成电路类：版图、工艺角、后仿真、时序分析、功耗分析

## 二、结构期望

### 2.1 标准章节结构

```
第1章 绪论
  1.1 研究背景与意义
  1.2 国内外研究现状
  1.3 主要研究内容与论文结构
第2章 相关理论基础
  2.x 涉及的核心理论/协议/标准
第3章 系统/算法总体设计
  3.1 需求分析与指标确定
  3.2 总体方案设计（含方案比选）
  3.3 关键模块设计
第4章 详细设计与实现
  4.x 各模块详细设计（含仿真/硬件实现）
第5章 测试与结果分析
  5.1 测试环境与方案
  5.2 测试结果与分析
  5.3 性能对比与讨论
第6章 总结与展望
```

### 2.2 必选章节

- **需求分析与指标确定**：必须有明确的、可量化的设计指标（如带宽、增益、误码率、功耗等）
- **方案设计与比选**：至少比较两种方案，给出选择依据
- **测试与验证**：必须有独立的测试章节，不可将测试结果散落在设计章节中

### 2.3 可选章节

- 电磁兼容(EMC)分析
- 可靠性与环境适应性分析
- 成本分析与产业化讨论

### 2.4 结构红线

- **无指标设计**：通篇无量化设计指标，仅有定性描述 → 结构不合格
- **测试缺失**：有设计无测试，或测试仅占不足一页 → 结构严重缺陷
- **理论堆砌**：理论基础章节超过全文40%，设计与实现内容不足 → 结构失衡

## 三、专项规则权重调整

基于通用六维度审查，对电子通信类论文的权重进行如下调整：

| 审查维度 | 默认权重 | 调整后权重 | 调整理由 |
|----------|----------|------------|----------|
| 引言/问题提出 | 1.0 | 1.0 | 保持不变，工程类论文同样需要清晰的问题定义 |
| 文献综述 | 1.0 | 0.9 | 略降，工程实践比文献覆盖更重要 |
| 方法说明 | 1.0 | **1.4** | 显著提升，设计方案的完整性和合理性是核心 |
| 分析论证 | 1.0 | **1.3** | 提升，测试数据的分析深度直接反映工程能力 |
| 结论建议 | 1.0 | 1.0 | 保持不变 |
| 事实描述 | 1.0 | **1.2** | 提升，技术参数和实验数据的准确性至关重要 |

## 四、方法审查补充维度

### 4.1 系统架构合理性

**审查要点：**

- 系统框图是否层次清晰，模块边界明确
- 模块划分是否遵循高内聚低耦合原则
- 接口定义是否完整（信号类型、电平标准、协议、时序）
- 是否有从功能需求到模块划分的映射关系说明
- 方案比选是否基于量化指标（而非仅凭"简单""成熟"等定性词）

**判定标准：**

- 合格：框图清晰、接口有定义、模块划分有依据
- 不合格：框图仅为示意、模块间信号流向不明、无接口规格

### 4.2 仿真实验规范

**审查要点：**

- 仿真工具选择是否合理并说明版本号
- 仿真模型假设条件是否明确列出
- 关键参数设置是否完整披露（步长、迭代次数、收敛阈值、网格大小等）
- 仿真结果是否经过收敛性验证（如网格无关性检验、步长敏感性分析）
- 仿真边界条件是否与实际场景一致

**常用仿真工具及审查侧重：**

| 工具 | 适用场景 | 重点审查项 |
|------|----------|------------|
| MATLAB/Simulink | 算法仿真、系统建模 | 采样频率设置、仿真步长、随机种子 |
| Multisim/PSpice | 电路仿真 | 元器件模型来源、温度参数、瞬态/稳态设置 |
| HFSS/CST | 电磁仿真 | 网格划分精度、边界条件类型、收敛判据 |
| ADS | 射频/微波电路 | S参数仿真频率范围、端口阻抗设置 |
| Vivado/Quartus | FPGA设计 | 时序约束、资源利用率、时钟域处理 |

### 4.3 硬件实现完整性

**审查要点——完整链路：**

原理图设计 → PCB布局布线 → 制板焊接 → 单元测试 → 整机联调

每个环节的审查标准：

- **原理图**：关键芯片选型有依据，电源设计有计算，去耦电容配置合理
- **PCB**：层叠结构说明，关键信号走线规则（阻抗匹配、等长等），设计规则检查(DRC)通过
- **实物**：有清晰的实物照片，标注关键模块位置
- **测试**：测试方案覆盖所有设计指标，测试仪器型号与精度明确

**必须包含项：**

- 核心元器件选型对比表（至少关键IC）
- BOM清单或关键器件列表
- 测试点定义与说明

### 4.4 算法性能评估

**审查要点：**

- 算法时间复杂度和空间复杂度分析是否给出
- 收敛性是否有理论证明或实验验证
- 是否与至少一种基线算法进行对比
- 对比条件是否一致（相同数据集、相同硬件平台、相同参数配置）
- 性能指标是否全面（不可仅报告单一指标）

**电子通信领域常用性能指标：**

- 通信系统：误码率(BER)、信噪比(SNR)、吞吐量、时延、频谱效率
- 信号处理：均方误差(MSE)、信噪比改善、处理延迟、计算复杂度
- 硬件系统：功耗、面积、最高工作频率、资源利用率
- 射频/微波：增益、带宽、回波损耗、噪声系数、线性度(IP3)

### 4.5 测试与验证

**审查要点：**

- 测试环境描述是否完整（仪器型号、校准状态、环境温度）
- 测试用例是否覆盖正常工况与极端工况
- 测试结果是否可重复（是否说明重复次数、统计方法）
- 仿真结果与实测结果是否有对比分析
- 偏差是否给出合理解释

**测试仪器引用规范：**

必须注明仪器品牌、型号、关键性能参数（如带宽、采样率、精度等级），例如："使用Keysight DSOX3024T示波器（带宽200MHz，采样率5GSa/s）"。

### 4.6 信号处理规范

**审查要点：**

- 采样定理是否遵守（采样率 ≥ 2倍信号最高频率），是否有抗混叠措施
- 频域分析是否标注频率分辨率、窗函数类型及选择依据
- 滤波器设计是否给出完整技术指标（通带/阻带频率、纹波、衰减）
- 数字信号处理中定点化是否讨论字长效应
- 时频分析方法选择是否与信号特性匹配

## 五、证据与论证标准补充

### 5.1 可接受的证据类型及效力排序

| 效力等级 | 证据类型 | 要求 |
|----------|----------|------|
| 最高 | 实物测试数据 | 注明仪器型号、测试条件、重复次数 |
| 高 | 后仿真/联合仿真数据 | 模型经过验证，参数完整披露 |
| 中 | 前仿真/行为级仿真 | 明确假设条件与简化说明 |
| 较低 | 理论推导/公式计算 | 推导完整，假设条件明确 |
| 最低 | 引用他人数据 | 注明出处，说明适用条件差异 |

### 5.2 证据红线

- 仿真结果不披露任何参数设置 → **证据无效**
- 硬件测试不注明仪器型号和测试条件 → **证据效力降至最低**
- 算法对比使用不同条件（如不同数据集、不同信噪比范围） → **对比结论无效**
- 性能声称"优于"但无统计显著性讨论 → **结论过度**
- 仿真截图无法辨认坐标轴数值 → **需要重新提供**

### 5.3 论证模式要求

- **系统设计型**：需求 → 指标 → 方案比选 → 详细设计 → 测试验证 → 指标达成度分析
- **算法研究型**：问题建模 → 算法推导 → 复杂度分析 → 仿真验证 → 基线对比 → 优劣讨论
- **仿真分析型**：建模假设 → 仿真配置 → 参数扫描 → 结果分析 → 模型验证 → 局限性讨论
- **硬件实现型**：选型依据 → 电路设计 → PCB实现 → 调试过程 → 测试数据 → 指标对照

## 六、常见问题模式

### 6.1 S级问题（严重，必须修改）

#### S-1：仿真黑箱

- **表现**：给出仿真结果图表，但不披露仿真工具版本、模型参数、边界条件、步长等任何设置细节
- **影响**：结果不可复现，无法判断结论是否可信，论文科学性根基丧失
- **修改方向**：补充完整的仿真配置表，包括工具名称与版本、所有关键参数值、收敛判据、运行环境

#### S-2：无可行性验证的系统设计

- **表现**：提出系统设计方案但未进行任何形式的验证（无仿真、无原型、无计算），仅停留在框图级别
- **影响**：无法证明方案可行，论文沦为设计构想而非工程实践
- **修改方向**：至少完成关键模块的仿真验证或原型测试，给出量化的性能数据

#### S-3：无基线对比的"优化"声称

- **表现**：标题或正文声称"优化""改进"某算法/系统，但全文无任何基线方法的对比数据
- **影响**：优化声称无证据支撑，违反基本学术规范
- **修改方向**：选取至少一种公认基线方法，在相同条件下进行对比实验，用数据量化改进幅度

#### S-4：关键设计指标缺失

- **表现**：整篇论文无明确的、可量化的设计目标，无法判定设计是否"成功"
- **影响**：缺乏评判标准使得测试与结论章节失去意义
- **修改方向**：在方案设计章节开头明确列出所有设计指标及目标值，结论章节逐项对照

### 6.2 A级问题（重要，需要修改）

#### A-1：缺少误差/不确定度分析

- **表现**：给出测试数据但不分析测量误差来源，不给出不确定度评定
- **影响**：读者无法判断数据可信区间，结论可靠性存疑
- **修改方向**：识别主要误差来源，给出测量不确定度评估或至少给出多次测量的统计量（均值、标准差）

#### A-2：仅有仿真无硬件验证讨论

- **表现**：硬件类课题仅完成仿真，对仿真与实际的差距无任何讨论
- **影响**：忽略寄生效应、加工偏差、环境影响等工程现实，结论适用性不明
- **修改方向**：若客观条件限制无法制作实物，须专门讨论仿真模型的局限性以及实际实现可能遇到的问题

#### A-3：框图与实现不一致

- **表现**：总体设计中的系统框图与后续详细设计章节的实际实现存在明显出入（模块增减、接口变化等）
- **影响**：论文内部逻辑矛盾，反映设计过程缺乏管控
- **修改方向**：统一框图与实现描述，若设计迭代导致变化须在文中说明变更原因

#### A-4：性能指标不完整

- **表现**：仅报告单一性能指标（如仅报告准确率），忽略其他关键维度（延迟、功耗、面积、复杂度等）
- **影响**：无法全面评价系统性能，可能掩盖设计短板
- **修改方向**：根据论文类型补充完整的性能指标集，至少覆盖该领域公认的核心指标

#### A-5：元器件选型无依据

- **表现**：直接给出所选芯片/器件型号，不说明选型过程和比较
- **影响**：读者无法判断选型合理性，降低设计说服力
- **修改方向**：对关键器件给出至少2-3种候选型号的对比表，从性能、成本、可获取性等维度说明选择理由

#### A-6：时序/时钟域问题未处理

- **表现**：FPGA或数字系统设计中未讨论时钟域交叉、时序约束、建立/保持时间
- **影响**：可能导致亚稳态等严重工程问题，反映设计能力不足
- **修改方向**：补充时序约束分析，跨时钟域信号须有同步处理方案说明

### 6.3 B级问题（建议改进）

#### B-1：图表质量不达标

- **表现**：仿真软件截图直接粘贴，分辨率低，坐标轴标注模糊或为英文默认标注
- **影响**：影响论文可读性和专业形象
- **修改方向**：使用矢量图或高分辨率导出，重新标注坐标轴（含中文说明、单位、字号统一）

#### B-2：公式单位不一致

- **表现**：同一公式中混用不同单位制（如dBm与W混用且未转换），或变量定义处与使用处单位不同
- **影响**：公式可信度降低，可能导致数值错误
- **修改方向**：全文统一使用国际单位制(SI)，必要的单位转换须显式给出

#### B-3：元器件选型理由缺失

- **表现**：非关键元器件（如电阻、电容值）的选取未说明计算依据
- **影响**：设计细节不够严谨
- **修改方向**：对影响性能的被动元件给出选值计算过程或引用设计手册推荐值

#### B-4：代码/HDL缺少注释

- **表现**：嵌入论文的关键代码段或HDL描述无注释，变量命名无意义
- **影响**：可读性差，无法验证逻辑正确性
- **修改方向**：添加功能注释，关键变量使用有意义的命名

#### B-5：仿真与实测偏差未量化

- **表现**：提到"仿真与实测基本一致"但未给出偏差数值
- **影响**：定性描述缺乏说服力
- **修改方向**：计算仿真与实测的相对误差/绝对误差，制作对比表格或叠加曲线图

## 七、文献综述特殊要求

### 7.1 文献来源优先级

| 优先级 | 来源类别 | 示例 |
|--------|----------|------|
| 最高 | 顶级期刊 | IEEE JSSC、IEEE TWC、IEEE TSP、电子学报 |
| 高 | 权威会议 | ISSCC、DAC、ICC、GLOBECOM、VTC |
| 中 | 一般SCI/EI期刊 | 通信学报、信号处理、电子与信息学报 |
| 较低 | 学位论文/专利 | 博士/硕士论文、发明专利 |
| 最低 | 教材/网络资源 | 仅作为背景知识引用 |

### 7.2 文献综述结构要求

- 按技术演进脉络组织，而非按发表时间简单罗列
- 每项关键技术至少引用3篇以上文献，展现研究谱系
- 须明确指出现有工作的不足，自然引出本文的研究切入点
- 最近3年的文献占比不低于30%（快速发展领域应更高）

### 7.3 审查侧重

- 是否覆盖了该方向的里程碑工作（如经典算法的原始论文）
- 是否区分了技术路线的不同流派并分别综述
- 是否引用了相关国际/国家标准（如3GPP、IEEE 802.x、国标GB/T等）
- 综述是否为后续的方案选择提供了铺垫和依据

## 八、术语与规范

### 8.1 缩写与术语

- 所有英文缩写首次出现时必须给出全称和中文翻译，格式：中文全称(English Full Name, ABBR)
- 示例：信噪比(Signal-to-Noise Ratio, SNR)
- 首次定义后，正文中统一使用缩写形式
- 同一概念全文术语必须一致，不可混用（如"载波"和"载波信号"应统一）

### 8.2 符号规范

- 变量使用斜体，单位使用正体
- 矢量使用粗体或加箭头，与标量明确区分
- 下标含义明确，物理量下标用正体（如 $V_{out}$、$f_{carrier}$）
- 常用物理常量使用标准符号（如 $c$ 为光速，$k_B$ 为玻尔兹曼常数）

### 8.3 单位规范

- 全文使用国际单位制(SI)
- 电子通信领域常用单位须规范书写：dBm、dBi、GHz、MHz、Mbps、Gbps
- 对数单位(dB)须标明参考值（如dBm以1mW为参考，dBW以1W为参考）
- 频率单位层级：Hz → kHz → MHz → GHz，选择使数值在1-999范围内的层级

### 8.4 图表规范

- 电路原理图须使用标准符号（参照GB/T 4728或IEEE标准）
- 仿真波形图须标注时间轴/频率轴单位、幅度单位、关键数据点
- PCB版图须标注尺寸、层数、关键信号走线
- 系统框图中信号流向须用箭头明确标注，模块间注明信号类型

### 8.5 参考文献格式

- IEEE期刊/会议论文须包含：作者, "标题," 期刊/会议名, vol., no., pp., 月 年, doi
- 标准文件须注明标准号、版本、发布年份
- 芯片数据手册(Datasheet)引用须注明制造商、器件型号、文档版本号

## 九、结论与建议特殊要求

### 9.1 结论章节必备内容

- **指标达成度对照表**：将设计初期确定的技术指标与最终测试结果逐项对比
- **关键设计决策回顾**：总结设计过程中的关键技术选择及其效果
- **局限性坦诚陈述**：明确指出设计中未能解决的问题或未达标的指标

### 9.2 展望章节要求

- 改进方向须具体，避免"进一步优化性能"等空泛表述
- 至少从以下维度之一给出展望：
  - 性能提升路径（如更高频率、更低功耗、更高集成度）
  - 功能扩展方向（如增加协议支持、扩展应用场景）
  - 工程化路径（如从原型到产品的关键步骤）
- 展望内容须与本文工作有逻辑关联，不可天马行空

### 9.3 审查红线

- 结论中出现论文正文未涉及的数据或结论 → **S级问题**
- 指标达成度完全不提 → **A级问题**
- 展望部分完全缺失或仅有一句话 → **B级问题**
